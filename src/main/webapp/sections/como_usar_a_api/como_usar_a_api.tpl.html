Como listar conjuntos de dados?

Para listar conjuntos de dados, a API deve ser consumida por meio de uma requisição HTTP GET para:

http://dados.ufpe.br/iago/open/list_datasets

Em resposta, a API retornará os seguintes valores em formato application/JSON:

datasetTitle Nome do Conjunto de Dados
keywords Palavras-chave que descrevem o conjunto de dados
collectorDataset Identificador do conjunto de dados para acessá-lo no portal
description Breve descrição do conjunto de dados
publisher O responsável pela publicação do conjunto de dados
category A categoria que o conjunto de dados se enquadra
views O total de visualizações do conjunto de dados

Como visualizar os metadados de conjuntos de dados?

Para listar os metadados em DCAT de um conjuntos de dados, a API deve ser consumida por meio de uma requisição HTTP GET para:

http://dados.ufpe.br/iago/open/dcat/{datasetTitle}

É necessário enviar o seguinte parâmetro:

datasetTitle Nome do Conjunto de Dados

Em resposta, A API retornará todos os metadados em formato application/JSON. A descrição dos metadados podem ser encontradas no Vocabulário DCAT.


Como consultar subconjuntos de dados?

Para consultar subconjuntos de dados, a API deve ser consumida. Para isso, é necessário enviar uma requisição HTTP POST em formato application/JSON para:

http://dados.ufpe.br/iago/open/execute_query

Os parâmetros a serem preenchidos são:

datasetName Nome do Conjunto de Dados em que o subconjunto deve se originar
version Versão do Conjunto de Dados que o subconjunto deve se originar
parameters Lista de parâmetros em que será realizada a filtragem, por ser em formato Map, é totalmente flexível para que possa receber qualquer propriedade para a verificação. A comparação deve ser feita apenas em chave-valor.
limit Limite máximo de registros a serem retornados. O valor máximo é de 10000.
page Página referente ao resultado obtido. Cada página contém 10000 registros. Recomenda-se que para a primeira requisição, o campo seja preenchido com o valor “1”.

Supondo que uma consulta seja realizada em um conjunto de dados hipotético "Alunos", em que o usuário deseje retornar alunos com idade de 20 anos do curso de Ciência da Computação: 

              {
                "datasetName" : "Alunos",
                "version" : "42891448230",
                "parameters" : {
                      "idade" : "20",
                      "curso" : "Ciência da Computação"
              },
                "limit" : "2000",
                "page" : 1
              }

Em resposta, A API retornará os seguintes valores em formato application/JSON:

datasetName Nome do Conjunto de Dados em que o subconjunto se originou
pagesNumber Número total de páginas do resultado da consulta. Cada página contém no máximo 10000 registros
rowsNumber Número total de registros do resultado da consulta
atualPage Página referente aos registros retornados
data Dados de retorno

Como recuperar todos os dados em formato desejado?

Para recuperar os dados em um formato específico, você deve consumir a API enviando uma requisição HTTP GET para:

http://dados.ufpe.br/iago/open/{datasetTitle}/format/{formato}/download

Em que os parâmetros {datasetTitle} e {formato} devem ser preenchidos:

datasetName Nome do Conjunto de Dados em que o subconjunto deve se originar
formato O formato físico do arquivo a ser recuperado (exemplo: XML, CSV, JSON)

Em resposta, a API exportarará o arquivo no formato especificado.

Como verificar as sugestões para os próximos conjuntos de dados?

Para recuperar as sugestões enviadas para os próximos conjuntos de dados, é necessário consumir a API enviando uma requisição HTTP GET para:

http://dados.ufpe.br/iago/open/list_suggestions

Em resposta, API retornará os seguintes valores em formato application/JSON: 

            {
              "description": "string",
              "email": "string",
              "name": "string",
              "solicitationDate": "2018-09-24T22:57:50.064Z",
              "status": "EVALUATION"
            }
description A descrição da solicitação enviada.
email O e-mail de quem solicitou, para tirar dúvidas ou enviar sugestões.
name O nome de quem solicitou
solicitationDate Data da solicitação
status O status indicando se foi aceito (ACCEPTED), se está em avaliação (EVALUATION) ou se não foi aceito (NOT ACCEPTED).

Como visualizar todas as versões disponíveis de um conjunto de dados?

Para visualizar todas as versões disponíveis de um conjunto de dados, a API deve ser consumida por meio de uma requisição HTTP GET para:

http://dados.ufpe.br/iago/open/{datasetTitle}/list_versions

É necessário enviar o seguinte parâmetro:

datasetTitle Nome do Conjunto de Dados

Em resposta, A API retornará todas as versões do conjuntos de dados em formato application/JSON. 

            {
              "date": "2018-09-24T22:57:50.064Z",
              "description": "string",
              "version": "string"
            }

Como recuperar um conjunto de dados em uma versão específica?

Para recuperar os dados em uma versão específica, você deve consumir a API enviando uma requisição HTTP GET para:

http://dados.ufpe.br/iago/open/{datasetTitle}/version/{versionId}

Em que os parâmetros {datasetTitle} e {versionId} devem ser preenchidos:

datasetName Nome do Conjunto de Dados em que o subconjunto deve se originar
versionId O identificador da versão desejada

Em resposta, a API permitirá a recuperação de todos os dados na versão especificada em formato application/JSON.
